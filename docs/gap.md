# Gap Analysis: Performance Review Module (HRPMS vs. Provided Sheet)

| #   | Feature / Aspect                            | Current System (HRPMS)                                                                                                                                                               | Provided Sheet (Herd Document)                                                                                                         | Gap / Remarks                                                                       |
| --- | ------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| 1   | **Performance Review Types**                | 3 types: Annual, Mid-term, Key Goal Review                                                                                                                                           | No mention of review types                                                                                                             | Sheet does not define or categorize review types                                    |
| 2   | **Fiscal Year Plan**                        | No provision for planning next fiscal year                                                                                                                                           | Includes plan for next fiscal year (2081/82)                                                                                           | System lacks next fiscal year planning section                                      |
| 3   | **Project-Based vs Person-Based**           | Review is person-based (multiple projects can be considered under one person for FY 2081/82)                                                                                         | Mentions project explicitly                                                                                                            | Sheet is project-centric; system is person-centric                                  |
| 4   | **Objectives / Key Goals**                  | Called "Key Goals"; employee adds their own, can view & validate last year's key goals                                                                                               | Called "Objectives"                                                                                                                    | Terminology differs; system has additional last-year goal validation feature        |
| 5   | **Key Goal Comments**                       | Both employee and line manager comment; line manager writes first, then also comments during approval                                                                                | Comments section present                                                                                                               | System has a dual-comment structure; sheet structure not clearly defined            |
| 6   | **Extra Keywords / Goals**                  | Provision to add extra keywords in performance review                                                                                                                                | Not explicitly mentioned                                                                                                               | Sheet lacks this feature                                                            |
| 7   | **Professional Development Progress**       | Users can view previous year key goals in Key Goal Review section                                                                                                                    | Progress on Professional Development Plan for FY 2081/82 mentioned                                                                     | System covers this via previous year key goals view                                 |
| 8   | **Training & Development Plan**             | Line manager writes a development plan answering: _"What training/development opportunities are needed in next 1 year..."_                                                           | Same question present in sheet                                                                                                         | Both have this; system assigns it to line manager                                   |
| 9   | **Manager Assessment (Results & Comments)** | No structured section separating "Results" and "Improvement Comments"; no enforcement of contribution impact, expectation comparison, underperformance areas, or improvement actions | Requires: key results description, contribution to team/org, comparison vs. expectations, 2–3 shortfall areas, improvement suggestions | **Major Gap** — System lacks structured, mandatory manager assessment fields        |
| 10  | **Core Competencies Rating**                | Rating scale 1–5 available                                                                                                                                                           | Rating scale present + requires examples reflecting the role                                                                           | System does not require written examples alongside ratings                          |
| 11  | **Next Period Goal Setting**                | System prompts user to identify at least 3 key goals for the next review period                                                                                                      | Has training/development opportunity question for next 1 year                                                                          | Approach differs: system focuses on goal-setting; sheet focuses on training needs   |
| 12  | **Manager Assessment Comments**             | Manager adds comments and acknowledgements                                                                                                                                           | Manager writes result comments based on review type                                                                                    | Sheet is more specific to review-type-based result comments; system is more general |
| 13  | **Final Comments Section**                  | Inputs from line managee, line manager, and reviewer (HR / Executive Director)                                                                                                       | Not clearly structured in sheet                                                                                                        | System has a more defined multi-level comment structure at the end                  |
| 14  | **Self Rating**                             | Manager rates the managee only; no self-rating by employee                                                                                                                           | Self-rating by employee is present                                                                                                     | **Gap** — System does not support employee self-rating                              |
| 15  | **Approval Workflow**                       | Line managee submits → Line manager approves → Final reviewer (HR / Executive Director) verifies                                                                                     | Approval process not clearly defined                                                                                                   | System has a structured 3-tier approval workflow; sheet lacks this clarity          |
| 16  | **Signature Feature**                       | No signature feature for any of the 3 approvers                                                                                                                                      | Likely implies physical/digital signature at approval                                                                                  | **Gap** — System does not support signatures in the approval process                |

---

**Summary of Key Gaps:**

| Priority | Gap                                                                    |
| -------- | ---------------------------------------------------------------------- |
| High     | No structured Manager Assessment section (Results + Improvement areas) |
| High     | No self-rating feature for employees                                   |
| Medium   | No next fiscal year planning section                                   |
| Medium   | No signature/acknowledgement feature in approval workflow              |
| Low      | Core competency section lacks example/evidence input                   |
| Low      | Sheet is project-centric while system is person-centric                |
